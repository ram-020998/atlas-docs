Installation Guide

Complete setup instructions for Appian Atlas

Quick Navigation: Python Setup MCP Server GitHub Token Install Power Configure Verify

Prerequisites

  • Kiro IDE installed
  • Python 3.10 or later
  • pip available in your terminal
  • A GitHub personal access token

Step 1: Verify Python Version

Appian Atlas requires Python 3.10 or later. Check your version:

python3 --version

If you have Python 3.9 or earlier, install Python 3.10+:

macOS
Windows
Linux
# Using Homebrew
brew install python@3.10

# Or download from python.org
# https://www.python.org/downloads/
# Download installer from python.org
# https://www.python.org/downloads/
# Make sure to check "Add Python to PATH" during installation
# Ubuntu/Debian
sudo apt update
sudo apt install python3.10

# Fedora
sudo dnf install python3.10

# Arch
sudo pacman -S python

Step 2: Install the MCP Server

Open the integrated terminal in Kiro IDE (Ctrl+` or Cmd+`).

Option A: Using pipx (Recommended)

pipx automatically creates an isolated environment:

pipx install "appian-atlas @ git+https://github.com/ram-020998/gam-knowledge-base.git#subdirectory=mcp_server"

Option B: Using pip with virtual environment

Create and activate a virtual environment first:

macOS/Linux
Windows
# Create virtual environment
python3 -m venv ~/.venvs/appian-atlas

# Activate it
source ~/.venvs/appian-atlas/bin/activate

# Upgrade pip
pip install --upgrade pip

# Install the package
pip install "git+https://github.com/ram-020998/gam-knowledge-base.git#subdirectory=mcp_server"
# Create virtual environment
python -m venv %USERPROFILE%\.venvs\appian-atlas

# Activate it
%USERPROFILE%\.venvs\appian-atlas\Scripts\activate

# Upgrade pip
pip install --upgrade pip

# Install the package
pip install "git+https://github.com/ram-020998/gam-knowledge-base.git#subdirectory=mcp_server"
Note: Always use a virtual environment or pipx. Do not install globally to avoid conflicts with other Python packages.

Verify Installation

appian-atlas --help

You should see:

usage: appian-atlas [-h] (--data-dir DATA_DIR | --github OWNER/REPO)
                     [--branch BRANCH] [--data-prefix DATA_PREFIX]
Troubleshooting: If appian-atlas is not found, use the full Python module path: 
python -m mcp_server --help

Step 3: Generate GitHub Token

The MCP server fetches data from GitHub at runtime, so it needs authentication.

  1. Go to GitHub ‚Üí Settings ‚Üí Developer settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
  2. Click "Generate new token (classic)"
  3. Give it a name (e.g., "Appian Atlas MCP")
  4. Select scope: repo (for private repos) or public_repo (for public repos)
  5. Click "Generate token" and copy it
Important: Save your token securely. You won't be able to see it again after leaving the page.

Installation Steps

  1. Open the Powers tab in Kiro IDE
  2. Click Add Power
  3. Paste the URL for your role
  4. Wait for installation to complete
Note: The SAIL Reference power doesn't require MCP server configuration - it's a pure documentation power.

Step 5: Configure GitHub Token

Add your GitHub token to the Power configuration:

  1. In Kiro IDE, go to the Powers tab
  2. Find your installed Appian Atlas power
  3. Click Open Power Config
  4. Find the line with "GITHUB_TOKEN": "${GITHUB_TOKEN}"
  5. Replace ${GITHUB_TOKEN} with your actual token:
"GITHUB_TOKEN": "ghp_your_token_here"
  1. Save the config
  2. Go to the MCP Servers panel in Kiro
  3. Find appian-atlas and click reconnect (or restart Kiro)
Security Tip: The token is stored locally in your Kiro configuration and is never shared.

Step 6: Verify Everything Works

  1. Open the Kiro sidebar and look for the MCP Servers section
  2. You should see appian-atlas listed with a green status indicator
  3. Open the Kiro chat and ask:
What applications are available?

You should get a response listing available applications with object counts and bundle stats.

Troubleshooting

401 Authentication Error

Symptom: MCP server shows 401 errors or fails to fetch data.

Solution:

  1. Go to Powers tab in Kiro
  2. Find your Appian Atlas power and click Open Power Config
  3. Verify your GitHub token is correctly set (not ${GITHUB_TOKEN})
  4. Save and reconnect the MCP server

MCP Server Not Found

Symptom: appian-atlas command not found after installation.

Solution:

# Use the Python module directly
python -m mcp_server --github ram-020998/gam-knowledge-base

# Or find the install location
pip show appian-atlas

Python Version Error

Symptom: Error about Python version requirement.

Solution: Install Python 3.10 or later (see Step 1) and create a new virtual environment with the correct Python version:

python3.10 -m venv ~/.venvs/appian-atlas

Server Shows as Disconnected

Solution:

  1. Check that Python 3.10+ is available: python3 --version
  2. Check that the package is installed: pip list | grep appian-atlas
  3. Try running the server manually to see errors:
appian-atlas --github ram-020998/gam-knowledge-base
  1. Click the reconnect button in the Kiro MCP Servers panel
